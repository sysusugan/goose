5 分钟上手 goose

goose 是一个可扩展的开源 AI agent，它会通过自动化编码任务来提升你的软件开发效率。

这份快速上手会带你完成：

  • ✅ 安装 goose
  • ✅ 配置你的 LLM
  • ✅ 创建一个小应用
  • ✅ 添加一个 MCP server

开始吧 🚀

安装 goose​

  • macOS
  • Linux
  • Windows

你可以选择安装 goose Desktop 和 / 或 goose CLI：

  • goose Desktop
  • goose CLI

点击下方按钮下载 goose Desktop：

macOS Apple 芯片 macOS Intel
  1. 解压下载的 zip 文件。
  2. 运行可执行文件启动 goose Desktop。

运行以下命令安装 goose：

curl -fsSL https://github.com/block/goose/releases/download/stable/download_cli.sh | bash

你可以选择安装 goose Desktop 和 / 或 goose CLI：

  • goose Desktop
  • goose CLI

点击下方按钮下载 goose Desktop：

DEB 安装包（Ubuntu / Debian） RPM 安装包（RHEL / Fedora） Flatpak（通用）

对于 Debian / Ubuntu 系发行版：

  1. 下载 DEB 文件
  2. 在终端进入该文件所在目录
  3. 运行 sudo dpkg -i (filename).deb
  4. 从应用菜单启动 goose

在 Linux 上安装 goose CLI，请运行：

curl -fsSL https://github.com/block/goose/releases/download/stable/download_cli.sh | bash

你可以选择安装 goose Desktop 和 / 或 goose CLI：

  • goose Desktop
  • goose CLI

点击下方按钮下载 goose Desktop：

Windows
  1. 解压下载的 zip 文件。
  2. 运行可执行文件启动 goose Desktop。

请在 Git Bash、MSYS2 或 PowerShell 中运行以下命令，以原生方式在 Windows 上安装 goose CLI：

curl -fsSL https://github.com/block/goose/releases/download/stable/download_cli.sh | bash

前置条件与环境要求请查看安装指南。

PATH 警告与 Keyring

如果安装后看到 PATH 警告，需要先把 goose 加入 PATH，再执行 goose configure。详细步骤见 Windows CLI 安装说明。

如果配置时提示你选择是否存储到 keyring，建议选择不存。如果遇到 keyring 报错，请继续参考 Windows 设置说明。

配置 Provider​

goose 需要依赖支持的 LLM providers来获得理解请求和执行任务所需的 AI 能力。首次使用时，系统会提示你配置一个 provider。

  • goose Desktop
  • goose CLI

在欢迎页中，你会看到这些选项：

  • Quick Setup with API Key - goose 会根据你的 API Key 自动识别并配置 Provider。
  • ChatGPT Subscription - 使用 ChatGPT Plus / Pro 账户登录，访问 GPT-5 Codex 模型。
  • Agent Router by Tetrate - 通过自动配置快速接入多个 AI 模型。
  • OpenRouter - 通过单一 API 接入多个模型，按量计费。
  • Other Providers - 在设置中手动配置其他 Provider。

对于这份快速开始，建议先选 Tetrate Agent Router。Tetrate 提供多个 AI 模型入口，内置 rate limiting 与自动故障切换。更多 provider 信息见配置 LLM Provider。

goose 会为你打开浏览器，完成 Tetrate 登录或注册。回到 goose Desktop 后，你就可以开始第一次会话了。

Provider 速率限制
不同 provider 的限流策略差异很大。开始时可以先选择你最熟悉、也最稳定的 provider。

如果你频繁遇到速率限制，可以参考处理速率限制这篇指南，或者切换模型，或改用支持自动故障切换的 provider。

goose relies heavily on tool calling capabilities and currently works best with Claude 4 models.

开始会话​

会话是你和 goose 之间的一段连续对话。现在开始创建一个。

  • goose Desktop
  • goose CLI

选择好 LLM provider 后，点击侧边栏中的 Home。

在输入框里直接输入问题、任务或指令，goose 就会立刻开始工作。

  1. 先创建一个空目录（例如 goose-demo），然后在终端进入该目录。
  2. 运行以下命令启动新会话：
goose session

编写 Prompt​

你可以直接像对开发者说话一样，把任务交给 goose。

现在让 goose 帮你做一个井字棋小游戏：

create an interactive browser-based tic-tac-toe game in javascript where a player competes against a bot

goose 会先生成计划，然后立即开始实现。完成后，你的目录里通常会出现一个 JavaScript 文件，以及一个可直接游玩的 HTML 页面。

启用一个扩展​

虽然你也可以手动进入工作目录，再自己在浏览器中打开 HTML 文件，但如果 goose 能直接帮你打开浏览器，不是更方便吗？现在启用 Computer Controller 扩展，让 goose 获得浏览器操作能力。

  • goose Desktop
  • goose CLI
  1. 点击左上角的 打开侧边栏。
  2. 点击侧边栏中的 Extensions。
  3. 启用 Computer Controller 扩展。这个扩展会提供网页自动化、文件缓存和自动化流程能力。
  4. 回到当前会话。
  5. 现在 goose 已经有浏览器能力了，让它在浏览器里打开你的游戏：
  1. 先按 Ctrl+C 结束当前会话，回到终端。
  2. 运行配置命令：
goose configure
  1. 依次选择 Add Extension > Built-in Extension > Computer Controller，并把 timeout 设为 300 秒。这个扩展会提供网页自动化、文件缓存和自动化流程能力。
┌   goose-configure
│

│  Add Extension
│
◇  What type of extension would you like to add?
│  Built-in Extension
│

│  Computer Controller
│

│  300

  1. 现在 goose 已经有浏览器能力了，恢复你上一轮会话：
 goose session -r
  1. 让 goose 在浏览器里打开你的游戏：
open the tic-tac-toe game in a browser

去玩一下你的小游戏吧，别装了，你肯定想点开试试 😂 祝你好运！

下一步​

恭喜，你已经成功用 goose 做出了一个 Web 应用！🎉

你可以继续这样探索：

  • 继续当前会话，把你的小游戏做得更完整（样式、功能、体验）
